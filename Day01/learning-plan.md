#### 90 Days DevOps Execution Blueprint



##### Current Level



I am a final-year IT student and full-stack developer.

I have built multiple web applications using ASP.NET Core, PHP, MySQL, and role-based authentication systems.

I understand application development but lack strong knowledge in Linux, CI/CD, Docker, Kubernetes, and Cloud infrastructure.



I am transitioning from Developer → DevOps Engineer.



My Understanding of DevOps \& Cloud Engineering



DevOps is not just tools — it is a culture of automation, ownership, collaboration, and continuous improvement between development and operations.



Cloud Engineering is about building scalable, reliable, and automated infrastructure using cloud platforms like AWS, Azure, or GCP.



A DevOps Engineer ensures:



Applications are deployed efficiently



Infrastructure is automated



Systems are reliable and scalable



CI/CD pipelines run smoothly



Failures are handled proactively



##### Why I Am Starting DevOps



I want to become a highly skilled, high-value engineer.



I want to deploy and manage real production systems.



I want financial growth and strong technical confidence.



I don’t want to depend only on development — I want end-to-end control.



##### Where I Want to Reach (90-Day Goals)



Deploy a production-grade application on Kubernetes (with CI/CD).



Build and automate infrastructure using AWS (EC2, S3, IAM, VPC basics).



Create a complete CI/CD pipeline (GitHub Actions/Jenkins) for one of my projects.



##### 3 Core DevOps Skills I Will Build



Linux \& Troubleshooting



Shell scripting



Process management



Logs \& debugging



Containerization \& Orchestration



Docker (images, volumes, networking)



Kubernetes (pods, services, deployments)



CI/CD \& Cloud



GitHub Actions / Jenkins



AWS deployment



Infrastructure basics



##### Time Commitment



Weekdays: 2–2.5 hours daily

 

Weekends: 4–6 hours



Total Weekly Target: ~18 hours



No zero days.



##### Consistency Plan



Study before distractions (fixed evening slot).



Push daily commits to GitHub (public accountability).



Share weekly progress on LinkedIn.



No overthinking — execute first, improve later.



If I miss one day → compensate next day (no excuses).



##### Execution Mindset



I will treat these 90 days like a job.

Discipline > Motivation.

Consistency > Intensity.

Execution > Perfection.



By the end of these 90 days, I will not just “learn DevOps” —

I will operate like a DevOps Engineer.
